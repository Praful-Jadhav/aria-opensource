# Contributing to ARIA Core

Thank you for your interest in contributing!

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/aria-core`
3. Follow the Quick Start in README.md
4. Create a branch: `git checkout -b feature/your-feature-name`

## Code Standards

- TypeScript strict mode
- ESLint must pass: `npm run lint`
- Build must pass: `npm run build`
- All API routes must have Zod validation
- All new services must follow the single-responsibility pattern

## Pull Request Process

1. One feature or fix per PR
2. Update documentation if you change behavior
3. Add tests for new functionality
4. Fill out the PR template completely
